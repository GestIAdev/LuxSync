feat(aether): WAVE 3505.2 ó Zero-Allocation Router implementation

Implementa los dos pilares de rendimiento del Motor Agnostico:
NodeGraph (ECS World) e IntentBus (hot-path bus) como clases
concretas zero-allocation, optimizadas para 44 Hz / 2000 nodos.

?????????????????????????????????????????????????????????????
NODEGRAPH.ts ó Data-Oriented ECS World
?????????????????????????????????????????????????????????????

Arquitectura en dense arrays por familia (COLOR/IMPACT/KINETIC/
BEAM/ATMOSPHERE). Iteracion O(N) cache-friendly garantizada.
V8 puede prefetch los proximos nodos antes de necesitarlos
(todos los objetos del array comparten la misma hidden class).

Indices multi-criterio pre-construidos en patch time:
  _slotIndex   Map<NodeId, {family, index}> ? O(1) acceso por ID
  _zoneIndex   Map<ZoneId, NodeId[]>        ? O(1) lookup geografico
  _roleIndex   Map<NodeRole, NodeId[]>      ? O(1) lookup semantico
  _deviceIndex Map<DeviceId, NodeId[]>      ? O(1) para unregister

Vistas tipadas (NodeView<T>) pre-creadas por familia.
getView() en el hot path: O(1), 0 allocaciones, 0 copias.
Los Systems reciben INodeView<IColorNodeData> type-safe.

Swap-and-pop en unregisterDevice: eliminar un nodo es O(1)
(sin desplazar el array, sin huecos). Los dense arrays
permanecen siempre compactos.

INTENTBUS.ts ó Zero-Allocation Hot-Path Router
?????????????????????????????????????????????????????????????

Buffer pre-allocated de 4096 IntentSlots (DEFAULT_CAPACITY).
Estrategia de zero-alloc para el campo values (Record<string,num>):
pool de 4096 objetos pre-allocados, reutilizados in-place
via for...in delete + for...in copy en push().

clear()             O(1) ó reset de dos enteros (writeHead, overflowed)
push()              O(1) ó property overwrite, 0 new objects
buildIndex()        O(N) ó algoritmo prefix-sum de dos pasadas:
  Pasada 1: cuenta intents por NodeId
  Pasada 2: rellena _nodeSlotBuffer con indices de slots
getIntentsForNode() O(1) lookup + O(k) copia de k punteros (k<=32)
  Resultado via _resultView (IntentSlotReadonlyView pre-allocated)

Estructuras de indice sobre TypedArrays (Int32Array):
  _nodeSlotBuffer[capacity]           indices de slots por nodo
  _nodeSlotListStart[MAX_UNIQUE_NODES] offsets de inicio por NodeId
  _nodeSlotListCount[MAX_UNIQUE_NODES] conteos por NodeId
  _nodeRangeBuffer[MAX_UNIQUE_NODES*2] buffer de rangos (legacy)
