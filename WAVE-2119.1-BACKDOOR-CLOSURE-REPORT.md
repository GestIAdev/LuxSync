## 🔴 EL PROBLEMA REAL

Después de **4 WAVEs fallidas** (2116→2119) intentando calibrar umbrales, pesos espectrales y coincidencia multi-banda, el BPM seguía bloqueándose en **161 BPM** con Boris Brejcha (~126 BPM real).

Evidencia del log `caoseffect.md`:
```
185bpm (raw=185) conf=0.92 kicks=32 intervals=[372,325,372,325,372,325]
161bpm (raw=161) conf=0.91 kicks=37 intervals=[372,325,372,325,372,464]
```

325ms = offbeat. 372ms = offbeat. **Ninguno es un kick real de Brejcha** (que debería ser ~476ms a 126BPM).

## 🔍 ROOT CAUSE: DOS PUERTAS, UNA SIN CANDADO

El `GodEarBPMTracker.process()` tenía **DOS caminos paralelos** para registrar kicks:

### Camino A — Ratio-Based (CALIBRADO ✅)
```typescript
const isPeak = ratio >= KICK_RATIO_THRESHOLD;      // 1.7x sobre media
const isRising = delta >= KICK_DELTA_THRESHOLD;     // 0.015 incremento mínimo
if (isPeak && isRising && !inKick && debounceOk) {  // ← 4 CONDICIONES
    kickDetected = true;
}
```
Calibrado durante WAVEs 2116-2117. Funciona.

### Camino B — External Kick (LA PUERTA TRASERA 🚫)
```typescript
if (externalKickDetected && debounceOk && !kickDetected) {  // ← SOLO 2 CONDICIONES
    kickDetected = true;  // SIN ratio check. SIN delta check.
}
```
Acepta kicks del `SlopeBasedOnsetDetector` de GodEarFFT. **CERO filtrado de calidad.**

### La Fórmula Contaminada
`GodEarFFT.ts` línea 1400:
```typescript
kickDetected = this.onsetDetector.detectOnset('kick', rawBands.subBass + rawBands.bass * 0.5)
```
- `subBass + bass * 0.5` = la misma mezcla contaminada que incluye offbeats
- `SlopeBasedOnsetDetector` threshold: `max(0.05, avgEnergy * 0.3)` = permisivo
- Resultado: **CADA transitorio de bajo es un "kick"**, incluyendo offbeats de Brejcha

### POR QUÉ WAVE 2118 y 2119 FALLARON
Ambas WAVEs modificaron `trackerEnergy` — la señal de entrada del **Camino A**.
Pero el **Camino B** bypaseaba TODO — inyectaba kicks directamente, sin importar qué fórmula tuviera Camino A.

```
WAVE 2118: subBass*1.5 + bass*0.4 → solo mejora Camino A → Camino B sigue contaminando
WAVE 2119: subBass*(1+beaterClick*5) → solo mejora Camino A → Camino B sigue contaminando
```

## ✅ LA SOLUCIÓN

**Cerrar la puerta trasera.** Punto.

El Camino B fue **deshabilitado** con comentario extensivo explicando por qué. La firma de `process()` sigue aceptando `externalKickDetected` para no romper el caller (`senses.ts`), pero lo ignora.

El tracker ahora usa **exclusivamente** su detección ratio-based (Camino A) con los umbrales calibrados en WAVE 2117:
- `KICK_RATIO_THRESHOLD = 1.7`
- `KICK_DELTA_THRESHOLD = 0.015`
