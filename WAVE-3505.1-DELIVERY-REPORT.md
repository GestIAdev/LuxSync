### 2. **capability-node.ts** (391 líneas)
**Responsabilidad:** Contratos de nodos de capacidad — la unidad fundamental del Aether

#### Contratos principales:

**INodeChannelDef** — Definición de un canal DMX individual
```ts
interface INodeChannelDef {
  type: AetherChannelType                // Qué es este canal (red, dimmer, pan, etc.)
  dmxOffset: number                      // Offset relativo al Device base
  defaultValue: number                   // Valor por defecto (0 o 128)
  is16bit?: boolean                      // ¿16-bit (coarse+fine)?
  customName?: string                    // Nombre custom para canales unknown
}
```

**INodeConstraints** — Restricciones físicas del hardware
```ts
interface INodeConstraints {
  minValue: number                       // Valor DMX mínimo absoluto
  maxValue: number                       // Valor DMX máximo absoluto
  responseMs: number                     // Tiempo de respuesta físico (ms)
  minStepSize?: number                   // Tamaño de paso mínimo (granularidad)
  maxSpeed?: number                      // Velocidad máxima de cambio
  safetyZoneMin?: number                 // Límite de seguridad inferior (p.e. tilt)
  safetyZoneMax?: number                 // Límite de seguridad superior
  calibrationPoints?: Array<{            // Puntos de calibración lineales
    dmxValue: number
    physicalValue: number
  }>
}
```

**ICapabilityNode** — Base atómica (sin discriminación)
```ts
interface ICapabilityNode {
  nodeId: NodeId                         // Identificador único
  deviceId: DeviceId                     // Qué Device lo contiene
  family: NodeFamily                     // Qué familia (COLOR, IMPACT, etc.)
  role: NodeRole                         // Semántica reactiva
  channels: readonly INodeChannelDef[]   // Array de canales que este nodo expone
  constraints?: INodeConstraints         // Restricciones físicas (opcional)
  position?: Position3D                  // Posición XYZ para IK (opcional)
  zone?: ZoneId                          // Qué zona geográfica del escenario
}
```

**5 interfaces discriminadas por NodeFamily:**

| Interface | Family | Propósito |
|---|---|---|
| **IColorNodeData** | COLOR | Emisión cromática: RGB, RGBW, CMY, rueda de colores, CTO/CTB, UV, Amber |
| **IImpactNodeData** | IMPACT | Interrupción de luz: dimmer, shutter, strobe (con "Snappy Attack" philosophy) |
| **IKineticNodeData** | KINETIC | Movimiento: pan, tilt, pan_fine, tilt_fine, speed, rotation |
| **IBeamNodeData** | BEAM | Conformación de haz: zoom, focus, iris, frost, gobo, prism, prism rotation |
| **IAtmosphereNodeData** | ATMOSPHERE | Atmósfera: pump, fan, hazzer, spark — controlado por cues, no frame |

**AnyNodeData** — Union discriminada para type safety
```ts
export type AnyNodeData =
  | IColorNodeData
  | IImpactNodeData
  | IKineticNodeData
  | IBeamNodeData
  | IAtmosphereNodeData
```

#### Características:

- ✅ **Discriminated unions** — Cada familia tiene su propia interface tipada
- ✅ **Extensible** — Nuevos canales se agregan sin pisar familias existentes
- ✅ **Safety-first** — Constraints capturan límites reales del hardware
- ✅ **Temporal semantics** — ResponseMs y MaxSpeed capturan dinámica física
- ✅ **Philosophy embedded** — "Snappy Attack" documentado en IImpactNodeData

**IDeviceDefinition** — La carcasa empaquetadora
```ts
interface IDeviceDefinition {
  deviceId: DeviceId
  nodeIds: readonly NodeId[]             // Nodos que este Device contiene
  dmxUniverse: number                    // Universo DMX (0-based)
  dmxBaseAddress: number                 // Dirección base en el universo (1-512)
  calibration?: IDeviceCalibration       // Datos de calibración opcional
}
```

#### Concepto clave:

> El Device es un empaquete logístico. Solo existe en dos momentos:
> 1. **Patch time** — La Forja lo crea y asigna universo+dirección
> 2. **Flush time** — El NodeResolver lo reagrupa para ensamblar paquetes DMX
> 
> Entre ambos, el motor solo ve nodos.

### 4. **node-graph.ts** (273 líneas)
**Responsabilidad:** La Matriz — registro central de todos los nodos (ECS pragmático)

#### Contratos principales:

**INodeView<T>** — Vista tipada zero-alloc sobre subconjunto de nodos
```ts
interface INodeView<T extends AnyNodeData> {
  readonly familyCount: number           // Cuántos nodos de esta familia
  readonly byId: ReadonlyMap<NodeId, T>  // Acceso por NodeId O(1)
  readonly byZone: ReadonlyMap<ZoneId, T[]>   // Acceso por Zone O(1)
  readonly byRole: ReadonlyMap<NodeRole, T[]> // Acceso por Role O(1)
  forEach(callback: (nodeId: NodeId, data: T) => void): void
}
```

**INodeGraph** — Interfaz principal
```ts
interface INodeGraph {
  // Registro de Devices
  registerDevice(device: IDeviceDefinition): void
  unregisterDevice(deviceId: DeviceId): void
  
  // Vistas tipadas para cada System
  getColorNodes(): INodeView<IColorNodeData>
  getImpactNodes(): INodeView<IImpactNodeData>
  getKineticNodes(): INodeView<IKineticNodeData>
  getBeamNodes(): INodeView<IBeamNodeData>
  getAtmosphereNodes(): INodeView<IAtmosphereNodeData>
  
  // Búsqueda multi-criterio
  getNodesByZone(zone: ZoneId): AnyNodeData[]
  getNodesByRole(role: NodeRole): AnyNodeData[]
  getNodesByFamily(family: NodeFamily): AnyNodeData[]
  
  // Export para diagnostics
  snapshot(): INodeGraphSnapshot
}
```

#### Características de diseño:

- ✅ **Dense arrays** — Iteración cache-friendly por familia
- ✅ **Zero-alloc views** — Cada System recibe un wrapper sin copias
- ✅ **O(1) lookups** — Multi-index by zone, role, family, device
- ✅ **Frame-safe immutability** — Estructura inmutable durante frame loop
- ✅ **Diagnostics snapshot** — Export completo para auditoría

### 5. **intent-bus.ts** (346 líneas)
**Responsabilidad:** Bus de alta velocidad entre Systems y Arbiter — cero allocations

#### Contratos principales:

**INodeIntent** — Unidad atómica de deseo
```ts
interface INodeIntent {
  nodeId: NodeId                         // Nodo objetivo
  values: Record<AetherChannelType, number>  // Canal → valor 0-1 normalizado
  priority: number                       // 0-99: Base, 100-199: Selene, 200-299: Manual, 300-399: Effects, 900+: Blackout

  }

```

**IIntentBus** — Pre-allocated array, zero-alloc
```ts
interface IIntentBus {
  readonly MAX_INTENTS_PER_FRAME: number

  clear(): void                          // Reset write pointer a 0

  push(intent: INodeIntent): void        // Write siguiente posición

  getIntentsForNode(nodeId: NodeId): readonly INodeIntent[]

  forEach(callback: (intent: INodeIntent) => void): void

  isEmpty(): boolean
  
  count(): number
}
```

**INodeArbiter** — Compositor multicapa
```ts
interface INodeArbiter {
  // Consume el bus y produce resolutivo arbitrado
  arbitrate(intents: IIntentBus): ArbitratedNodeMap

  // Utilidades
  getStrategy(channel: AetherChannelType): MergeStrategy
}
```

**INodeResolver** — Intents → DMX
```ts
interface INodeResolver {
  // Aplica calibration + transfer curves + convierte a DMX 0-255
  resolve(
    arbitratedNodes: ArbitratedNodeMap,
    graph: INodeGraph
  ): IDMXPacket[]
}
```

#### Concepto clave — Zero-alloc hot path:

```
Per frame:
  1. intents.clear()                    // Reset write ptr (0 alloc)
  2. for each System: intents.push()    // Write to pre-alloc slots (0 alloc)
  3. arbiter.arbitrate(intents)         // In-place merge (0 alloc)
  4. resolver.resolve(arbitrated)       // DMX conversion (0 alloc)
  
Total GC pressure: 0 allocations
```
