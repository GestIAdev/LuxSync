## FASE 2: THE HAL UPGRADE ✅

### ColorTranslator.ts — REESCRITO COMPLETO (480 líneas)

#### VULN-COLOR-01: Weighted RGB → CIE L*a*b* ΔE*

**Antes:**
```
weightedRgbDistance = √((ΔR×0.30)² + (ΔG×0.59)² + (ΔB×0.11)²)
→ BT.601 luminance coefficients in RGB space (perceptualmente incorrectos)
```

**Después:**
```
Pipeline: sRGB → Linear → XYZ (D65) → L*a*b* → CIE76 ΔE*
ΔE*ab = √((ΔL*)² + (Δa*)² + (Δb*)²)
→ Perceptualmente uniforme. La fotocopiadora murió.
```

Funciones implementadas:
- `srgbToLinear(c)` — IEC 61966-2-1 gamma decode
- `rgbToXyz(rgb)` — sRGB primaries → XYZ con D65 illuminant matrix
- `xyzToLab(xyz)` — XYZ → CIE L*a*b* (D65 white: Xn=0.95047, Yn=1.0, Zn=1.08883)
- `rgbToLab(rgb)` — convenience chain
- `deltaE76(lab1, lab2)` — CIE76 perceptual distance

**¿Por qué CIE76 y no CIEDE2000?**
CIE76 es perceptualmente uniforme *enough* para matching de 8-14 colores de rueda mecánica.
CIEDE2000 añade rotation terms chroma-dependent y es 10x más complejo para ganancia marginal
a esta granularidad. Si algún día necesitamos matching continuo fino, upgradeamos. Ahora no.

#### VULN-COLOR-02: RGBW Decomposition

```typescript
export function rgbToRgbw(rgb: RGB): RGBW {
  const w = Math.min(rgb.r, rgb.g, rgb.b)  // achromatic portion
  return { r: rgb.r - w, g: rgb.g - w, b: rgb.b - w, w }
}
```

- Tipo `RGBW` exportado: `{ r, g, b, w }` (0-255 each)
- `ColorTranslationResult` extendido con `rgbw?: RGBW`
- `translate()` detecta `mixing: 'rgbw'` y retorna RGBW decomposed

#### VULN-COLOR-03: CMY Subtractive

```typescript
export function rgbToCmy(rgb: RGB): CMY {
  return { c: 255 - rgb.r, m: 255 - rgb.g, y: 255 - rgb.b }
}
```

- Tipo `CMY` exportado: `{ c, m, y }` (0-255 each)
- `ColorTranslationResult` extendido con `cmy?: CMY`
- `translate()` detecta `mixing: 'cmy'` y retorna CMY converted

#### Cache Upgrade: FIFO → LRU con quantización perceptual

**Antes:** Cache key = `RGB(r,g,b):profileId` — quantizado en RGB con step=8

- `wheelLabCache`: Map<profileId, Lab[]> — pre-computa L*a*b* de colores de rueda una sola vez

## FASE 4: HALF-COLOR POSITIONING ✅ (VULN-COLOR-04)

### Problema
Las ruedas de color mecánicas permiten posicionamiento analógico: si Red está en DMX 15
y Orange en DMX 30, enviar DMX 22 posiciona la rueda *entre ambos*, creando una mezcla
óptica física. Pero el translator solo hacía snap al color más cercano — saltos bruscos.

### Solución
`findNearestColorLab()` ahora busca los **dos colores más cercanos** en L\*a\*b\*.
Si son adyacentes en la rueda y el target cae genuinamente entre ambos:

```
t = distanceToNearest / (distanceToNearest + distanceToSecond)
interpolatedDmx = round(dmxA + (dmxB - dmxA) × t)
```

**Guards:**
- Solo si ΔE\* al más cercano > 3 (no aplicar si ya es match perfecto)
- Solo si ΔE\* al segundo < 60 (segundo color debe ser relevante)
- Solo si son adyacentes en la rueda (index diff = 1 o wrap-around)
- El label muestra `Red/Orange` cuando t > 0.15

**Resultado:** Transiciones graduales de color en ruedas mecánicas. La rueda del
Beam 2R ahora se posiciona con precisión sub-ranura.
