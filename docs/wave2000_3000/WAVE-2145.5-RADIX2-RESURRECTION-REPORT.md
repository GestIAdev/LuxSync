## ANATOMÍA DE LA FALLA SPLIT-RADIX

### El Error Fundamental

Split-Radix DIF se basa en una **descomposición asimétrica**:

$$N = \underbrace{\frac{N}{2}}_{\text{evens}} + \underbrace{\frac{N}{4}}_{\text{odd evens}} + \underbrace{\frac{N}{4}}_{\text{odd odds}}$$

Sin embargo, la implementación iterativa usaba:

```typescript
for (let m = n; m > 2; m >>= 1) {  // ← Radix-2 topology (halving)
  // Pero el algoritmo espera Split-Radix topology (N/2+N/4+N/4)
}
```

### El Problema de Permutación

Split-Radix DIF requiere una permutación de salida **no estándar** ("digit-reversal"), no la simple bit-reversal de Radix-2. La implementación estaba haciendo:

```typescript
// Post-computation bit-reversal (incorrecto para Split-Radix DIF)
for (let i = 0; i < n; i++) {
  const j = bitRev[i];
  if (j > i) swap(outReal[i], outReal[j]);
}
```

Pero Split-Radix DIF necesita una permutación completamente diferente, que depende de cómo se interceptan los sub-DFTs.

### Prueba del Concepto: Recursive Split-Radix

Para probar que el **math** de Split-Radix era correcto, implementamos una versión recursiva directa:

```typescript
function recursiveSplitRadixDIT(x: Float32Array, stride: number, N: number): Complex[] {
  if (N === 1) return [{ re: x[0], im: 0 }];

  const even = recursiveSplitRadixDIT(x, stride * 2, N / 2);
  const oddEven = recursiveSplitRadixDIT(x.subarray(stride), stride * 4, N / 4);
  const oddOdd = recursiveSplitRadixDIT(x.subarray(3 * stride), stride * 4, N / 4);

  const result = new Array(N);
  const quarter = N / 4;
  
  for (let k = 0; k < quarter; k++) {
    const w1 = twiddle(2 * k, N);
    const w3 = twiddle(6 * k, N);

    result[k] = even[k] + w1 * oddEven[k] + w3 * oddOdd[k];
    result[k + quarter] = even[k + quarter] + w1 * oddEven[k + quarter] + w3 * oddOdd[k + quarter];
    result[k + N / 2] = even[k + N / 2] + w1 * oddEven[k] - w3 * oddOdd[k];
    result[k + 3 * quarter] = even[k + N / 2] + w1 * oddEven[k + quarter] - w3 * oddOdd[k + quarter];
  }
  return result;
}
```

**Resultado:** ✅ ALL TESTS PASSED (error ~ 1e-9)

**Conclusión:** La matemática de Split-Radix es correcta. El problema es la **conversión a iterativo**.

## ITERACIÓN 4: COOLEY-TUKEY RADIX-2 DIT (VICTORY)

### El Cambio Radical

Abandonamos completamente Split-Radix y volvimos a **Cooley-Tukey Radix-2 DIT** — el algoritmo más probado en la historia:

```typescript
function computeFFTCore(
  samples: Float32Array,
  outReal: Float32Array,
  outImag: Float32Array
): void {
  const n = samples.length;

  // STEP 1: Bit-reversal permutation of input
  const bits = Math.log2(n) | 0;
  for (let i = 0; i < n; i++) {
    let rev = 0, x = i;
    for (let b = 0; b < bits; b++) {
      rev = (rev << 1) | (x & 1);
      x >>= 1;
    }
    outReal[i] = samples[rev];
    outImag[i] = 0;
  }

  // STEP 2: Bottom-up DIT butterfly stages
  for (let size = 2; size <= n; size <<= 1) {
    const halfSize = size >> 1;
    const angleStep = -2 * Math.PI / size;

    for (let groupStart = 0; groupStart < n; groupStart += size) {
      for (let j = 0; j < halfSize; j++) {
        const angle = angleStep * j;
        const wr = Math.cos(angle);
        const wi = Math.sin(angle);

        const evenIdx = groupStart + j;
        const oddIdx = groupStart + j + halfSize;

        const tRe = wr * outReal[oddIdx] - wi * outImag[oddIdx];
        const tIm = wr * outImag[oddIdx] + wi * outReal[oddIdx];

        outReal[oddIdx] = outReal[evenIdx] - tRe;
        outImag[oddIdx] = outImag[evenIdx] - tIm;
        outReal[evenIdx] = outReal[evenIdx] + tRe;
        outImag[evenIdx] = outImag[evenIdx] + tIm;
      }
    }
  }
}
```

### ¿Por qué Radix-2 DIT?

1. **Simplicidad:** La topología es trivial — halving en cada stage
2. **Robustez:** Décadas de implementaciones probadas
3. **Permutación:** Bit-reversal es exacto y fácil
4. **Performance:** 0.564ms — bien dentro del budget
5. **Aritmética:** O(N log N) multiplicaciones complejas — óptimo
