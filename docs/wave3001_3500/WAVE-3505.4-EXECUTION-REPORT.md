### ✅ Tarea 2: Implementar `NodeResolver.ts` (~320 líneas)

**Arquitectura:**
```typescript
export class NodeResolver implements INodeResolver {
  private readonly _universeBuffers = new Map<number, Uint8Array>()  // Pre-alloc
  private readonly _activeUniverses = new Set<number>()              // Frame tracking
  private readonly _packetPool: MutableDMXPacket[] = []               // Output pool (pre-alloc)
  private readonly _framePackets = new Map<number, MutableDMXPacket>() // Frame output

  constructor(graph: INodeGraph) { ... }
  
  registerUniverse(universe: number): void
  resolve(arbitrated: ArbitratedNodeMap): readonly IDMXPacket[]
  getUniverseBuffer(universe: number): Uint8Array | undefined        // NEW: zero-copy
  get registeredUniverses(): IterableIterator<number>                 // NEW: iteration
}
```

**Flujo de `resolve(arbitrated)`:**
1. **Zero-fill:** `buf.fill(0)` todos los universos (native memset, rápido)
2. **Escribir nodos:**
   - Para cada `[nodeId, channelValues]` en `arbitrated`
   - Obtener `IDeviceDefinition` via `NodeGraph.getDevice()` (O(1))
   - Obtener `ICapabilityNode` via `NodeGraph.getNodeData()`
   - Para cada canal:
     - Aplicar **TransferCurve** (si existe)
     - Escalar normalizado [0,1] → DMX [0,255]
     - Aplicar **calibración física**
     - Clamp final a `[0, constraints.maxValue]`
     - Escribir en buffer (coarse byte, +fine byte si 16-bit)
3. **Emitir packets:** Ensamblar `IDMXPacket[]` desde buffers activos

**TransferCurves implementadas:**

| Tipo | Fórmula | Uso |
|------|---------|-----|
| `linear` | $x$ | Sin transformación |
| `exponential` | $x^{exponent}$ (default 2.5) | Para respuesta no-lineal (típicamente diodos) |
| `logarithmic` | $\frac{\ln(1+x)}{\ln(2)}$ | Respuesta inversa (sensibilidad alta en bajos) |
| `scurve` | $3x^2 - 2x^3$ (Hermite) | Smooth S-curve, suaviza transientes |
| `gamma` | $x^{1/\gamma}$ (default 2.2) | Corrección gamma (típicamente optics) |

**Calibración física:**

```typescript
// PAN (invertPan, panOffset)
if (invertPan) v = 255 - v
if (panOffset) v = v + offset

// TILT (invertTilt, tiltOffset, tiltLimitMin/Max)
if (invertTilt) v = 255 - v
if (tiltOffset) v = v + offset
v = clamp(v, tiltLimitMin, tiltLimitMax)

// DIMMER (dimmerScale)
dmxValue = Math.round(dmxValue * dimmerScale)
```

**16-bit channels:**
- Slot coarse: `(raw16 >> 8) & 0xFF` (MSB)
- Slot fine: `raw16 & 0xFF` (LSB)
- Ambos escritos en orden: coarse[idx], fine[idx+1]

**Zero-alloc hot path:**
- ✅ No `new Array()` en `resolve()`
- ✅ No `new Map()` en `resolve()`
- ✅ No `new Uint8Array()` en `resolve()`
- ✅ Object mutados in-place desde pools pre-allocated
- ✅ GC amortizado: buffers + pools crecen en patch-time, se estabilizan

#### 8d. Bloque en `processFrame()` (hot-path)

**Ubicación:** Justo después de `this.hal.flushToDriver(fixtureStates)` (legacy completo)

```typescript
// ═══════════════════════════════════════════════════════════════════════════
// ⚛️ WAVE 3505.4: AETHER MATRIX — V2 Agnostic Engine Pipeline
//
// Runs AFTER the legacy pipeline to not interfere with it.
// The _aetherBus receives intents from Systems in a future version.
// For now the NodeArbiter will arbitrate what it has (empty = default values).
// The pipeline is ready for each System to inject its intents.
//
// Zero-alloc: the Uint8Array buffers are owned by NodeResolver.
// They are sent to the driver by reference (zero-copy to hardware).
// ═══════════════════════════════════════════════════════════════════════════
if (this._aetherHasDevices && this.hal) {
  // 1. Clear the intent bus from the previous frame
  this._aetherBus.clear()

  // 2. Systems write their intents to _aetherBus
  //    (Systems will connect here in WAVE 3505.5+ as fixtures are migrated.
  //    For now the bus is empty and nodes emit defaultValues.)

  // 3. The Arbiter unifies all layers → ArbitratedNodeMap
  this._aetherArbiter.setSystemIntents(this._aetherBus)
  const arbitrated = this._aetherArbiter.arbitrate()

  // 4. NodeResolver translates to Uint8Array(512) per universe (pre-alloc, in-place)
  this._aetherResolver.resolve(arbitrated)

  // 5. Send to the DMX driver directly (zero-copy — uses Resolver's buffers)
  for (const universe of this._aetherResolver.registeredUniverses) {
    const rawBuf = this._aetherResolver.getUniverseBuffer(universe)
    if (rawBuf) this.hal.sendUniverseRaw(universe, rawBuf)
  }
}
```

## 🏗️ ARQUITECTURA FINAL

### Frame Loop de 44Hz (23ms interval)

```
FrameScheduler(23ms)
    ↓
processFrame() [async, Stampede Guard in FrameScheduler]
    ├─── [LEGACY PIPELINE]
    │    ├─ context = brain.getCurrentContext()
    │    ├─ engineAudioMetrics = {bass, mid, high, energy, ...}
    │    ├─ beatState = beatDetector.tick(now)
    │    ├─ intent = await engine.update(context, engineAudioMetrics)
    │    ├─ masterArbiter.setTitanIntent(...)
    │    ├─ effectManager.getCombinedOutput() → EffectIntents
    │    ├─ masterArbiter.setEffectIntents(...)
    │    ├─ arbitratedTarget = masterArbiter.arbitrate()
    │    ├─ fixtureStates = hal.renderFromTarget(arbitratedTarget, ...)
    │    ├─ hephRuntime.tick(now) → Hephaestus merging (if active)
    │    └─ hal.flushToDriver(fixtureStates) → IDMXDriver.send()
    │
    └─── [AETHER V2 PIPELINE] — NEW, runs AFTER legacy
         ├─ _aetherBus.clear()
         ├─ [Systems inject intents — ready for WAVE 3505.5]
         ├─ _aetherArbiter.arbitrate() → ArbitratedNodeMap
         ├─ _aetherResolver.resolve(arbitrated) → fills Uint8Array(512)
         └─ for each universe:
            └─ hal.sendUniverseRaw(universe, buffer) → IDMXDriver.sendUniverse() → Hardware
```

**Coexistence:** Ambos pipelines corren cada frame. El legacy sirve a fixtures tradicionales, Aether sirve a capability-node devices.

## 📝 LESSONS LEARNED

1. **Pre-allocation is paramount:** The difference between 780-byte hot path (today) and 22KB+/sec (naïve approach) is order-of-magnitude performance.

2. **Zero-copy abstractions:** `getUniverseBuffer()` → direct Uint8Array reference avoids copying 512 bytes × N universes × 44fps.

3. **Parallel coexistence:** Legacy + V2 pipelines can run simultaneously. No rip-and-replace. Incremental fixture migration.

4. **Calibration belongs in resolver:** Physical properties (invertPan, tiltLimits, etc.) are specific to devices and must live with the address translation logic.

5. **TransferCurves are codec-like:** They map perceptual intent (linear values from Systems) to hardware reality (non-linear DMX response).
