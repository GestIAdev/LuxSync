## Comportamiento implementado
- El proyector itera `FixtureState[]` y para cada fixture cuyo universo esté `claimed` por Aether (comprobado con `aetherConfig.ownsUniverse(fixture.universe + 1)`) aplica mutaciones in-place basadas en los nodos del `NodeGraph` del device correspondiente.
- Conversión de escalas: valores Aether normalizados 0-1 → 0-255 para el legacy UI (multiplicar por 255 y redondear/truncar).

### Mapeos más importantes (resumen)
- Color (IColorNode) → `fixture.r`, `fixture.g`, `fixture.b`
- Impact (IImpactNode) → `fixture.dimmer`
- Kinetic (IKineticNode) → `fixture.pan`, `fixture.tilt`

Fragmentos clave del método `project()` (extracto):

```ts
// COLOR → RGB
fixture.r = (currentColor.r * 255 + 0.5) | 0
fixture.g = (currentColor.g * 255 + 0.5) | 0
fixture.b = (currentColor.b * 255 + 0.5) | 0

// KINETIC → PAN/TILT
fixture.pan  = (currentPosition.pan  * 255 + 0.5) | 0
fixture.tilt = (currentPosition.tilt * 255 + 0.5) | 0
```

Regla de oro: el proyector NO escribe a `physicalPan`/`physicalTilt` (propiedad que controla la `FixturePhysicsDriver`) y respeta los campos `readonly` de la capa física.
