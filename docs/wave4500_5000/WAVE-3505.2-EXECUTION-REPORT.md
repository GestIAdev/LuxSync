## Implementaciones Entregadas

### 1. NodeGraph.ts — Data-Oriented ECS World

**Líneas de código:** 526 LOC (completo con comentarios de optimización).

**Arquitectura:**

```
┌─ DENSE ARRAYS (cache-friendly iteration)
│  ├─ _color[]       FamilyStore<IColorNodeData>
│  ├─ _impact[]      FamilyStore<IImpactNodeData>
│  ├─ _kinetic[]     FamilyStore<IKineticNodeData>
│  ├─ _beam[]        FamilyStore<IBeamNodeData>
│  └─ _atmosphere[]  FamilyStore<IAtmosphereNodeData>
│
├─ INDICES (constructed once at patch time)
│  ├─ _slotIndex     Map<NodeId, {family, index}>        O(1) lookup by ID
│  ├─ _zoneIndex     Map<ZoneId, NodeId[]>               O(1) geographic
│  ├─ _roleIndex     Map<NodeRole, NodeId[]>             O(1) semantic
│  └─ _deviceIndex   Map<DeviceId, NodeId[]>             O(1) for unregister
│
└─ VIEWS (pre-created per family, 0-alloc in hot path)
   ├─ _colorView      NodeView<IColorNodeData>
   ├─ _impactView     NodeView<IImpactNodeData>
   ├─ _kineticView    NodeView<IKineticNodeData>
   ├─ _beamView       NodeView<IBeamNodeData>
   └─ _atmosphereView NodeView<IAtmosphereNodeData>
```

**Decisiones de optimización:**

| Decisión | Razón | Beneficio |
|----------|-------|----------|
| Dense arrays por familia | CPU prefetcher ↑ | Cache-line optimization |
| Array<T> no TypedArray | Objetos JS discriminados | V8 inline caching (hidden class) |
| Map para índices | String NodeIds no numéricos | O(1) amortizado hash lookup |
| NodeView wrapper ligero | Cero copias de datos | 0 bytes allocados, 0 GC pressure |
| Swap-and-pop unregister | Mantiene arrays compactos | O(1) eliminación, sin huecos |

**Complejidades en hot-path (44 Hz):**

```
getView(family)        → O(1) table switch + return pre-existing instance
getNodesByZone(zoneId) → O(1) Map.get()
getNodesByRole(role)   → O(1) Map.get()
getDeviceNodes(devId)  → O(1) Map.get()
snapshot()             → O(N) off-path (debug only)
```

**Garantía de compactness:**
- Dense arrays nunca tienen huecos.
- Swap-and-pop en O(1): `store[targetIdx] = store[last]; store.pop()`.
- El slot moved se actualiza en _slotIndex también en O(1).

### 2. IntentBus.ts — Zero-Allocation Hot-Path Router

**Líneas de código:** 521 LOC (completo con TypedArray management).

**El problema central:**

Al principio parecía trivial, pero el campo `values: Record<string, number>` fue el reto:
- Si creamos `new {}` en cada `push()`, generamos 44 × 2000+ allocations/segundo.
- V8 GC no puede seguir el ritmo → micro-pauses visibles (flicker de luz).

**Solución adoptada:**

```typescript
// PRE-ALLOCATE un pool de 4096 objetos {}
_valuePool: Record<string, number>[] = new Array(4096)

// En push(), reusar el objeto del pool in-place
const valuesTarget = this._valuePool[idx]
// Limpiar keys previas (para reutilizar el objeto)
for (const key in valuesTarget) {
  delete valuesTarget[key]
}
// Copyar las nuevas keys del intent
for (const key in intent.values) {
  valuesTarget[key] = intent.values[key]
}
```

**Arquitectura del buffer:**

```
FRAME N:
┌─────────────────────────────────────────────────────────────┐
│ _slots[0..maxIdx] — Array<IntentSlot> pre-allocated        │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ {nodeId: 'dev1:color', values: {r:0.8, g:0.2}, ...}  │ │
│ │ {nodeId: 'dev2:impact', values: {dimmer:0.95}, ...}   │ │
│ │ {nodeId: 'dev1:color', values: {r:0.7, g:0.3}, ...}   │ │  ← Intents dispersos
│ │ ...                                                      │ │
│ └─────────────────────────────────────────────────────────┘ │
│ _writeHead = count de intents escritos                      │
└─────────────────────────────────────────────────────────────┘

INDEXING (buildIndex):
┌─ Pasada 1: Contar intents por NodeId
├─ Pasada 2: Prefix-sum para cal offsets, rellena _nodeSlotBuffer
│
└─ Resultado: _nodeRangeIndex Map<NodeId, rangeIdx>
             _nodeSlotBuffer [idx_slot_0, idx_slot_1, idx_slot_2, ...]
             _nodeSlotListStart [offset_0, offset_1, ...]
             _nodeSlotListCount [count_0, count_1, ...]

RETRIEVAL (getIntentsForNode):
┌─ O(1): Map.get(nodeId) → rangeIdx
├─ start = _nodeSlotListStart[rangeIdx]
├─ count = _nodeSlotListCount[rangeIdx]
└─ Copiar _slots[_nodeSlotBuffer[start..start+count]] a _resultBuffer
   Retornar _resultView (readonly wrapper, 0-alloc)
```

**Complejidades verificadas:**

```
clear()             O(1) — writeHead = 0, overflowed = false
push(intent)        O(1) — property overwrite, for...in on pre-alloc objects
buildIndex()        O(N) — two-pass prefix-sum algorithm
getIntentsForNode() O(1) + O(k) where k ≤ 32 (típicamente 1-5)
                    → O(1) Map.get() + O(k) pointer copy
forEach()           O(N) — iteration on raw buffer (off-path friendly)
```

**Pre-allocations in constructor:**

```
_slots[]            4096 × IntentSlot (~120 bytes each) → ~480 KB
_valuePool[]        4096 × Record<string,number>       → ~50 KB
_nodeSlotBuffer[]   Int32Array[4096]                    → ~16 KB
_nodeRangeBuffer[]  Int32Array[4096]                    → ~16 KB
_nodeSlotListStart  Int32Array[2048]                    → ~8 KB
_nodeSlotListCount  Int32Array[2048]                    → ~8 KB
_resultBuffer[]     IntentSlot[32]                      → ~4 KB
───────────────────────────────────────────────────────────────
Total heap resident: ~582 KB (one-time, never freed)
```

**Garantía HARD:** 0 allocations durante `clear()` → `push()` → `buildIndex()` → `getIntentsForNode()`.
